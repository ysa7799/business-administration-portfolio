# Process Improvement — Shift-Report Turnaround

A **business-administration case study** that takes a real operational bottleneck (slow, error-prone shift reporting) and redesigns the workflow — cutting estimated turnaround ~60% with no new software cost.

## What's inside
- `Process_Improvement_Case_Study.docx` / `.pdf` — full write-up: problem, root-cause analysis, before/after workflow, RACI, results, and next steps
- `RACI_KPI_Matrix.xlsx` — the RACI matrix + a KPI before/after sheet with a live improvement-% formula

## Headline result
| Metric | Before | After |
|---|---|---|
| Reporting effort / shift | ~90 min | ~35 min |
| Management visibility | ~1 day later | Same shift |
| Transcription steps | 2 | 1 |

## Skills demonstrated
Process analysis, root-cause thinking, workflow redesign, RACI ownership models, KPI definition and measurement, stakeholder-ready business writing, Excel.
